## Highlights

- Release assets now keep only meaningful deliverables and include the release tag in the file name.
- GitHub Release bodies now render from repository-owned release notes instead of GitHub
  auto-generated text alone.
- The top-level README and self-hosted guide now prioritize operators, and the repository ships
  Linux and Windows relay bootstrap installers with auto-start support.

## Included Iterations And Remediations

- Remediation v3 R1: release assets were reduced to real installers and runtime binaries, removing
  bundled repository README files and versionless wrapper names.
- Remediation v3 R2: release-note governance moved into `docs/releases/` with a render script wired
  into the release workflow.
- Remediation v3 R3: user-facing onboarding was rewritten across `README.md`, `README.en.md`, and
  `docs/self-hosted.md`, with Linux `systemd` and Windows startup-task installers added.

## Operator Notes

- Linux relay bootstrap now writes `/etc/vibe-relay/relay.env` and a `systemd` service.
- Windows relay bootstrap now writes a PowerShell env file plus launcher and registers a startup
  scheduled task.
- Public relay URL, token, and preview host remain runtime configuration. The install scripts do
  not hardcode them.
- macOS relay one-click installation is still not shipped because the repository does not yet
  publish macOS relay assets.
