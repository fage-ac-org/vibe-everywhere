## Highlights

- Windows CLI packaging, smoke staging, and installer flow now preserve the EasyTier runtime files
  beside the shipped executables instead of assuming raw binaries are enough.
- CI and Release now both treat GitHub-hosted Linux overlay smoke as a blocking gate, using a
  harness-only EasyTier `no_tun` path that validates truthful hosted-runner fallback behavior
  without changing product defaults.
- Cross-platform delivery verification is now aligned across Windows smoke, hosted Linux overlay
  gating, Android packaging, release-note rendering, and tagged asset publishing.

## Included Iterations And Remediations

- Iteration v2: delivery verification hardening across Windows, Android, and hosted Linux workflow
  coverage
- Iteration v3: workflow normalization and release-flow audit after the hosted Linux overlay repair
- Remediation v5: truthful overlay readiness and smoke alignment
- Remediation v6: Windows EasyTier runtime packaging alignment
- Remediation v7-v9: hosted Linux overlay harness stabilization, EasyTier `no_tun` adaptation, and
  blocking-gate restoration

## Operator Notes

- No operator-facing runtime defaults changed in this release. EasyTier `no_tun` remains a
  harness-only hosted-runner workaround and is not enabled by default in shipped relay or agent
  deployments.
- On Windows, keep the packaged runtime files such as `Packet.dll`, `wintun.dll`, and
  `WinDivert64.sys` beside `vibe-relay.exe` and `vibe-agent.exe` after extraction.
