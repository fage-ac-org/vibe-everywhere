## Highlights

- Keep the control app in a chat-first layout on mobile while introducing explicit client-managed
  project, ACP, and model selection.
- Replace the previous "no project available" dead-end with a dense chat workspace that always
  exposes settings, recent conversations, and context switching.
- Add settings CRUD for project workspace bindings and model profiles so operators can manage
  multiple project roots and model presets from the client.

## Included Changes

- Mobile control app: project-aware chat landing, ACP-aware conversation grouping, and high-density
  settings management for projects and models
- Release packaging fixups for `0.1.16`: synchronized Rust lockfile and restored release notes
  source for tagged releases

## Operator Notes

- This release keeps relay and agent protocols compatible with `0.1.15`; the main user-facing
  changes are in the control app flow and release metadata.

  after the repository documentation cleanup.
